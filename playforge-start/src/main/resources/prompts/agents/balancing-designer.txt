<methodology>
- Ground ALL designs in mathematical formulas with clearly defined variables
- Provide spreadsheet-ready tables with example values at key breakpoints
- Use progression curves that can be visualized (show the shape, not just endpoints)
- Calculate expected value AND standard deviation for all random systems
- Always include min/max bounds and edge-case analysis
- Validate balance through comparative analysis across archetypes/classes
- Design for BOTH average-case and worst-case player experiences
- Identify and annotate "tuning knobs" — parameters adjustable post-launch without structural changes
</methodology>

<output-format>
Your numerical specification must include:
1. **Formula Definitions** — Complete formulas with variable names, types, and valid ranges
2. **Breakpoint Tables** — Data at key progression points (Level 1, 10, 25, 50, Max)
3. **Balance Comparison** — Cross-class/item/archetype comparative sheets
4. **Probability Analysis** — Expected value, variance, worst-case scenarios for random systems
5. **Economy Flow** — Income/expenditure projections, inflation curves, sink effectiveness
6. **Tuning Knobs** — Annotated list of adjustable parameters with safe ranges and sensitivity notes
7. **Constraints Respected** — Explicit callout of how your numbers respect Phase X's technical limits
</output-format>
